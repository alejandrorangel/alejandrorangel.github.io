Books and articles that have shaped how I think about engineering, AI, and building things. I'll keep this updated as I go.

---

- [**Clean Code**](https://www.goodreads.com/book/show/3735293-clean-code) — Robert C. Martin <span class="tag tag-engineering">Engineering</span>
  <br/>Opinionated but useful. Good foundations for writing code that other people can actually read.

- [**Design Patterns**](https://www.goodreads.com/book/show/85009.Design_Patterns) — Erich Gamma, Richard Helm, Ralph Johnson, John Vlissides <span class="tag tag-engineering">Engineering</span>
  <br/>The Gang of Four classic. Dense but worth it — once you see these patterns, you spot them everywhere.

- [**The Algorithm Design Manual**](https://www.goodreads.com/book/show/425208.The_Algorithm_Design_Manual) — Steven Skiena <span class="tag tag-engineering">Engineering</span>
  <br/>More practical than CLRS. The war stories and catalog of algorithmic problems make it a great reference.

- [**Designing Data-Intensive Applications**](https://www.goodreads.com/book/show/23463279-designing-data-intensive-applications) — Martin Kleppmann <span class="tag tag-engineering">Engineering</span>
  <br/>The best book I've read on distributed systems. Changed how I think about data pipelines and system design.

- [**Designing Autonomous AI**](https://www.goodreads.com/book/show/61395461-designing-autonomous-ai) — Kence Anderson <span class="tag tag-ai">AI/ML</span>
  <br/>Practical guide to designing AI agents and autonomous systems. Relevant to the LLM tooling work I do daily.

- [How Claude Code Builds a System Prompt](https://www.dbreunig.com/2026/04/04/how-claude-code-builds-a-system-prompt.html) — Drew Breunig <span class="tag tag-ai">AI/ML</span>
  <br/>Great breakdown of context engineering — how the system prompt is assembled and why it matters.

- [**Generative Deep Learning**](https://www.goodreads.com/book/show/44144492-generative-deep-learning) — David Foster <span class="tag tag-ai">AI/ML</span>
  <br/>Solid intro to generative models. Good balance of theory and hands-on code.

- [**The Manager's Path**](https://www.goodreads.com/book/show/33369254-the-manager-s-path) — Camille Fournier <span class="tag tag-leadership">Leadership</span>
  <br/>Essential if you're going from IC to tech lead. Practical and honest about what each step looks like.

- [**Staff Engineer**](https://www.goodreads.com/book/show/56481725-staff-engineer) — Will Larson <span class="tag tag-leadership">Leadership</span>
  <br/>What does impact look like beyond senior? This book lays out the archetypes and trade-offs.

- [**The Design of Everyday Things**](https://www.goodreads.com/book/show/840.The_Design_of_Everyday_Things) — Don Norman <span class="tag tag-design">Design</span>
  <br/>Foundational HCI book. You'll never look at a door handle the same way again.

- [**Don't Make Me Think**](https://www.goodreads.com/book/show/18197267-don-t-make-me-think-revisited) — Steve Krug <span class="tag tag-design">Design</span>
  <br/>Short, practical, and still relevant. Good reminder that simplicity wins.
